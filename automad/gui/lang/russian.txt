sys_cache_info: Кэширование необходимо для ускорения работы вашего сайта, и его обычно следует включать. Однако при некоторых обстоятельствах вы можете захотеть отключить или очистить кеш.

-

sys_cache_clear: Очистить кэш

-

sys_cache_clear_info: Очистка кеша сайта приводит к повторному сканированию обновленных страниц и файлов. Это действие не удалит файлы страниц или изображения из каталога кеша.

-

sys_cache_monitor: Интервал проверки составляет

-

sys_cache_monitor_info: Когда кэширование включено, Automad будет регулярно проверять наличие изменений в файловой системе. Здесь можно определить частоту этих проверок.

-

sys_cache_lifetime: Время жизни кеша

-

sys_cache_lifetime_info: Все кэшированные данные и страницы Automad имеют максимальный срок службы. По истечении срока жизни объекта его кэш будет перестроен по запросу.

-

sys_cache_purge: Очистить каталог кеша

-

sys_cache_purge_info: В отличие от простой очистки кеша, очистка каталога кеша также перемещает все временные файлы, удаленные страницы, резервные копии обновлений и кэшированные изображения во временный каталог системы. Это вызовет полную перестройку кеша изображений и страниц, а также очистку всех удаленных страниц. В большинстве систем временный каталог будет очищаться автоматически время от времени или при перезагрузке. Однако в некоторых системах этот каталог необходимо очищать вручную.

-

sys_config: Редактировать файл конфигурации

-

sys_config_warning: Обратите внимание, что только опытные пользователи должны редактировать файл конфигурации, чтобы избежать повреждения вашей установки. Если что-то пойдет не так, пожалуйста, удалите файл конфигурации вручную с вашего сервера, чтобы восстановить настройки по умолчанию.

-

sys_debug_info: Когда отладка включена, все процессы Automad будут регистрироваться в консоли вашего браузера. Отладка необходима только для разработки или устранения неполадок и должна быть отключена во всех остальных случаях.

-

sys_headless_edit_info: Структура возвращаемых данных JSON контролируется шаблоном. Вы можете редактировать этот шаблон с помощью встроенного языка шаблонов Automad для настройки полей с помощью переменных, объектов и структур управления.

-

sys_headless_info: Активация безголового режима превращает Automad в систему управления контентом, полностью сосредоточенную на создании контента без уровня представления. Хотя панель мониторинга по-прежнему можно использовать для управления контентом, все данные и файлы доступны через API JSON только для чтения.
