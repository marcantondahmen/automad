sys_cache_info: La mise en cache est essentielle pour accélérer votre site et doit généralement être activée. Cependant, dans certaines circonstances, vous souhaiterez peut-être désactiver ou vider votre cache.

-

sys_cache_clear: Vider le cache

-

sys_cache_clear_info: La suppression du cache du site force une nouvelle analyse des pages et des fichiers mis à jour. Cette action ne supprimera aucun fichier de page ni aucune image du répertoire de cache.

-

sys_cache_monitor: L'intervalle de vérification est

-

sys_cache_monitor_info: Lorsque la mise en cache est activée, Automad vérifie régulièrement les changements dans le système de fichiers. La fréquence de ces contrôles peut être définie ici.

-

sys_cache_lifetime: La durée de vie du cache est

-

sys_cache_lifetime_info: Toutes les données et pages Automad mises en cache ont une durée de vie maximale. Une fois la durée de vie d'un objet dépassée, son cache sera reconstruit sur demande.

-

sys_cache_purge: Purger le répertoire du cache

-

sys_cache_purge_info: Contrairement à la simple suppression du cache, la purge du répertoire du cache déplacera également tous les fichiers temporaires, les pages supprimées, les sauvegardes de mise à jour et les images mises en cache vers le répertoire temporaire du système. Cela forcera une reconstruction complète de l'image et du cache de page ainsi qu'un nettoyage de toutes les pages supprimées. Sur la plupart des systèmes, le répertoire temporaire sera nettoyé automatiquement de temps en temps ou au redémarrage. Cependant, sur certains systèmes, ce répertoire doit être effacé manuellement.

-

sys_config: Modifier le fichier de configuration

-

sys_config_warning: Notez que seuls les utilisateurs expérimentés doivent modifier le fichier de configuration pour éviter d'endommager votre installation. En cas de problème, supprimez manuellement le fichier de configuration de votre serveur afin de restaurer les paramètres par défaut.

-

sys_debug_info: Lorsque le débogage est activé, tous les processus d'Automad seront enregistrés dans la console de votre navigateur. Le débogage n'est nécessaire que pour le développement ou le dépannage et doit être désactivé dans tous les autres cas.

-

sys_headless_edit_info: La structure des données JSON renvoyées est contrôlée par un modèle. Vous pouvez modifier ce modèle à l'aide du langage de modèle intégré d'Automad pour personnaliser les champs à l'aide de variables, d'objets et de structures de contrôle.

-

sys_headless_info: L'activation du mode sans tête transforme Automad en un système de gestion de contenu entièrement axé sur la création de contenu sans couche de présentation. Bien que le tableau de bord puisse toujours être utilisé pour gérer le contenu, toutes les données et tous les fichiers sont accessibles via une API JSON en lecture seule.
